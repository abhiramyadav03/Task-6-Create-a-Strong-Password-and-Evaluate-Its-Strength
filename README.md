# 📌 Project Overview

Passwords are the first line of defense for protecting digital systems and personal data. Weak passwords are one of the most common causes of security breaches, allowing attackers to gain unauthorized access through techniques such as brute-force attacks and dictionary attacks.

In this project, multiple passwords with different levels of complexity were created and tested using an online password strength analyzer. The results demonstrate how password length, randomness, and character diversity directly impact password security.

- Understand what makes a password strong.
- Evaluate password strength using an online password checker.
- Compare weak and strong passwords.

# 🔍 Password Attacks

## 1️⃣ Brute Force Attack

A brute-force attack systematically tries every possible password combination until the correct password is found.

### Characteristics

- Tries every possible combination
- Time-consuming
- Effective against short passwords
- Prevented by long and complex passwords

---

## 2️⃣ Dictionary Attack

A dictionary attack uses a predefined list of commonly used passwords and dictionary words instead of trying every possible combination.

Examples:

```
password
admin
welcome
qwerty
football
123456
```

### Characteristics

- Faster than brute force
- Highly effective against common passwords
- Easily defeated by unique and random passwords

# 🛡 Password Security Best Practices

- Use at least **12–16 characters**.
- Mix uppercase and lowercase letters.
- Include numbers and special characters.
- Avoid dictionary words.
- Avoid names and birthdays.
- Never reuse passwords.
- Enable Multi-Factor Authentication (MFA).
- Store passwords securely using a password manager.
- Change passwords immediately after a data breach.
